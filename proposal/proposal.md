## 專題動機

本組於上學期完成「海事勤務值勤管理系統」雛形，採用 PHP + MySQL 架構運行於 Windows XAMPP 環境，已具備登入、值勤打卡、請假申請與審核、三層級權限管理等功能。然而原系統存在三項主要不足：第一，部署仰賴 Windows XAMPP 環境，無法跨平台移轉，亦不利長期維運；第二，系統僅累積值勤紀錄而未有任何分析能力，管理層無法從歷史資料掌握人力分布、工作負荷、海象與值勤量之關聯等趨勢；第三，未具備海事業務特性，僅為通用打卡系統，與專題之海事領域定位脫節。

本學期目標為延續並強化此系統，將其改造為具備雲端容器部署能力、海事業務特性、資料分析洞察之整合性平台，符合本課程「巨量資料與雲端運算」之主題精神。

## 系統架構

系統採三容器架構，由 Docker Compose 統一編排。

第一容器為 Web 服務（`php:8.2-apache`），承載上學期 PHP 系統並完成 Linux 化改造，提供登入、值勤、請假、管理介面與分析儀表板頁面。第二容器為資料庫（`mysql:8.0`），存放使用者、值勤、請假與海象資料表，以環境變數管理連線設定。第三容器為分析服務（`python:3.11`），執行 Pandas 資料清洗、Matplotlib/Seaborn 圖表生成、以及 Gradio 互動式分析介面，分析結果以 PNG 與 HTML 形式輸出至共用 volume，由 PHP 端嵌入顯示。三容器共用 Docker 內部網路，PHP 與 Python 皆連向同一 MySQL 服務。

資料流程為：使用者操作 PHP 介面 → 寫入 MySQL → Python 分析容器讀取 MySQL → 產出視覺化圖表與 Gradio 儀表板 → 回饋至 PHP 顯示層。

## 預期成果

本專題完成一套可於 Linux 環境以單一 `docker compose up` 指令啟動之海事勤務值勤管理系統，整合 PHP 操作介面、MySQL 資料庫、Python 分析模組與 Gradio 互動儀表板。預計產出至少五張具洞察力之分析圖表，包含月度值勤時數趨勢、各海域值勤分布、海況與工作負荷關聯、請假類型分析、各角色出勤統計，並提供互動式儀表板供管理層即時查詢。

系統具備跨平台部署、可橫向擴充之特性，後續可延伸接入即時 AIS 船舶資料、行動裝置介面、操作稽核紀錄與班表自動排程等模組。
